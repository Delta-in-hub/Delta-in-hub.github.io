顶层/底层const

简单说，顶层const是指“指针本身是常量”，而底层const是指“指针指向的对象是常量”。

说白了就是常量指针和指向常量的指针。@陈伟的回答 - 知乎 https://www.zhihu.com/question/24785843/answer/128871816

关于top-level const，从标准角度来说，标准用的词叫做top-level cv-qualifier。 cv-qualifier 就是 const, volatile, const volatile，或者empty。

对于一个类型T（例如typedef const char * T;)，类型 cv T的top-level cv-qualifier就是cv。cv T的含义就是T是cv。

const T的top-level cv-qualifier就是const。含义是类型T是const的。
const volatile T的top-level cv-qualifier就是const volatile。含义是类型T是const volatile的。

注意的是，如果T 是const char *，那么 const T 其实等同于 const char * const 。大家都解释很多了，不详述了。

C++标准中并没有low-level const这种说法，是该书作者为了解释自己造出的词。@hearts zh的回答 - 知乎 https://www.zhihu.com/question/24785843/answer/128752641

constexpr 变量 :

  1. 其类型必须是字面类型 (LiteralType) 。
  2. 它必须被立即初始化
  3. 其初始化的全表达式，包括所有隐式转换、构造函数调用等，都必须是常量表达式

constexpr 函数 :

  1. 其返回类型（若存在）必须是字面类型 (LiteralType)
  2. 其每个参数都必须是字面类型 (LiteralType)
  3. 若函数或函数模板的任何声明拥有 constexpr 说明符，则每个声明必须都含有该说明符
  4. … (C++14极大的拓宽了限制)

从特性上讲，一个函数标不标constexpr，对于运行期的使用根本没有影响，只是声明可以编译期求值而已。

#include <bits/stdc++.h>

using namespace std;

int g_constructCount     = 0;
int g_copyConstructCount = 0;
int g_destructCount      = 0;

struct A
{
    A()
    { // 基本构造
        cout << "construct: " << ++g_constructCount << endl;
    }

    A(const A& a)
    { // 拷贝构造
        cout << "copy construct: " << ++g_copyConstructCount << endl;
    }

    ~A()
    { // 析构
        cout << "destruct: " << ++g_destructCount << endl;
    }
};

A getA()
{
    A a; // 第一次构造
    return a;
    // return A();  // 等价，分开写是为了便于说明
}

int main()
{
    A a2   = getA(); // 非右值引用
    A&& a3 = getA(); // 右值引用
    return 0;
}

非右值引用，关闭返回值优化(-fno-elide-constructors)

construct: 1        // 第一次构造，getA() 中的局部变量 a
copy construct: 1   // 第二次构造，将 a 复制给一个临时变量
destruct: 1           // 析构局部变量 a
copy construct: 2   // 第三次构造，将临时变量复制给 a2
destruct: 2           // 析构临时变量
destruct: 3           // 程序结束，析构变量 a2

右值引用，关闭返回值优化

construct: 1        // 第一次构造，getA() 中的局部变量 a
copy construct: 1   // 第二次构造，将 a 复制给一个临时变量
                    // 右值引用 a3 延长了临时变量的声明周期，使其没有马上被析构
destruct: 1           // 析构局部变量 a
destruct: 2           // 程序结束，析构变量 a3

开启RVO/NRVO (默认开启,-O0情况下也是开启的)

construct: 1        // 构造局部变量 a，在编译的优化下，相当于直接将 a “改名” aa
destruct: 1         // 程序结束，析构变量 aa

tmp在返回语句中隐式的被视为右值,编译器首先会进行 return-value-optimization (no copy, no move), 如果编译器发现不能RVO,则会调用移动构造函数,如果移动构造函数也没有,就会调用拷贝构造函数.

I.e. just as you would in C++03. tmp is implicitly treated as an rvalue in the return statement. It will either be returned via return-value-optimization (no copy, no move), or if the compiler decides it can not perform RVO, then it will use vector’s move constructor to do the return. Only if RVO is not performed, and if the returned type did not have a move constructor would the copy constructor be used for the return.

引用折叠规则

  1. 所有的右值引用叠加到右值引用上仍然还是一个右值引用。（T&& && 变成 T&&）
  2. 所有的其他引用类型之间的叠加都将变成左值引用。 （T& &, T& &&, T&& & 都变成 T&）
  3. 对常量引用规则一致

https://zh.cppreference.com/w/cpp/language/value_category

每个表达式都具有某种非引用类型，且每个表达式只属于三种基本值类别中的一种：纯右值 (prvalue)、亡值 (xvalue)、左值 (lvalue)。

  • 拥有身份的表达式被称作“泛左值 (glvalue) 表达式”。左值和亡值都是泛左值表达式。

    • 拥有身份 (identity)：可以确定表达式是否与另一表达式指代同一实体，例如通过比较它们所标识的对象或函数的（直接或间接获得的）地址；

  • 可被移动的表达式被称作“右值 (rvalue) 表达式”。纯右值和亡值都是右值表达式。

    • 可被移动：移动构造函数、移动赋值运算符或实现了移动语义的其他函数重载能够绑定于这个表达式。

  • 左值(lvalue)

    • 变量、函数或数据成员的名字，不论其类型，例如 std::cin 或 std::endl
    • 即使变量的类型是右值引用，由其名字构成的表达式仍是左值表达式 (右值引用本身也是左值表达式 )
    • 返回类型为左值引用的函数调用或重载运算符表达式,例如std::vector::operator[]
    • a = b，a += b，a %= b以及所有其他内建的赋值及复合赋值表达式
      • a = b这个表达式是左值,因为a = b = c = 2是合法的.
    • ++a , --a，内建的前置自增与前置自减表达式；
    • *p，内建的间接寻址表达式
    • a[n] 和 n[a]，内建的下标表达式
    • …
    • 字符串字面量，例如 "Hello, world!"
      • 可以理解为const左值类型

  • 纯右值(prvalue)

    • （除了字符串字面量之外的）字面量，例如 42、true 或 nullptr
    • 返回类型是非引用的函数调用或重载运算符表达式，例如 str.substr(1, 2)、str1 + str2 或 it++
    • a + b、a % b、a & b、a << b，以及其他所有内建的算术表达式；
    • a && b、a || b、!a，内建的逻辑表达式；
    • a < b、a == b、a >= b 以及其他所有内建的比较表达式；
    • &a，内建的取地址表达式；
    • …
    • 转换为非引用类型的转型表达式，例如 static_cast<double>(x)、std::string{} 或 (int)42；
    • this 指针；
    • 枚举项;
    • lambda 表达式

  • 亡值(xvalue)

    • 返回类型为对象的右值引用的函数调用或重载运算符表达式，例如 std::move(x)；
    • …

    • 从C++17开始, 返回值优化才被强制要求. 当然大部分编译器早已经实现,但还是有不会生效的情况.
    • 当编译器无法知道返回哪个实例的时候. i.e 条件判断,有多个return
    • 返回参数或者全局变量
    • 返回std::move()
      • Returning by calling std::move() on the return value is an anti-pattern. It is wrong most of the times. It will indeed attempt to force move-constructor, but in doing so it will disable RVO. It is also redundant, as move will happen if it can even without explicitly calling std::move() (see here).
    • 赋值 (operator=)
      • RVO can only happen when an object is created from a returned value. Using operator= on an existing object rather than copy/move constructor might be mistakenly thought of as RVO
    • 返回成员变量

RVO和移动语义是相辅相成的,不可因为一个否定另一个.

https://stackoverflow.com/questions/29859796/c-auto-vs-auto

  • auto
    • 无视引用与CV(const and volatile)限定符
    • Choose auto x when you want to work with copies.
    • For example, if you have const int& f(){...} then auto x = f(); deduces x as int, and not const int&.
  • auto &
    • 不会去掉CV限定符,推导的类型可能会是const限定的
    • 始终得到引用类型
    • Choose auto &x when you want to work with original items and may modify them. 如果不想修改,使用cosnt auto&
  • auto &&
    • 万能引用,可以绑定到左值或者右值
    • 不会去掉CV限定符,推导的类型可能会是const限定的

如果想要推导出原有的值类型与CV限定符,使用decltype(auto)

decltype

https://zh.cppreference.com/w/cpp/language/decltype

  • 希望从表达式的类型推导出变量的类型,但是不想用该表达式的值初始化变量

    • 在此过程中,编译器分析表达式并得到他的类型,却不实际计算表达式的值

  • 如果是个变量，那么类型就是声明这个变量时的类

    • decltype(variable)结果只有当variable本身是引用时才是引用

  • 如果是个表达式(将变量加括号也看作表达式)

    • 若 表达式 的值类别为亡值，则 decltype 产生 T&&；
    • 若 表达式 的值类别为左值，则 decltype 产生 T&；
    • 若 表达式 的值类别为纯右值，则 decltype 产生 T。

  • decltype((variable)) (注意是双层括号) 的结果永远是引用
